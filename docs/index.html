Type Coercion

Implicit type coercion follows C semantics:

// int <- float: truncation
int x;
x = 3.14;    // x == 3

// float <- int: promotion
float y;
y = 10;      // y == 10.0

// Mixed arithmetic: promote to float
3 + 2.5       // -> 5.5

Division Semantics

  • int / int → floor division: 7 / 2 = 3
  • float / int → float division: 7.0 / 2 = 3.5

Truthiness Rules

Follows C semantics: 0 and 0.0 are false; all other values are true.

I/O: printf

printf(42);         // prints: 42
printf(3.14);       // prints: 3.14
printf(x + y);      // prints computed sum

Comments

int x;   // this is a comment

Block Scoping

Blocks introduce new lexical scopes. Variables declared inside are destroyed when the block exits.

int x;
x = 10;

{
    int y;           // y exists only here
    y = 20;
    int x;           // shadows outer x
    x = 99;
    printf(x);       // prints: 99
}

printf(x);           // prints: 10

Variable Shadowing

An inner scope may declare a variable with the same name as an outer scope.

Stack-based Symbol Table

Efficient scope enter/exit with O(d) lookup where d = nesting depth.
